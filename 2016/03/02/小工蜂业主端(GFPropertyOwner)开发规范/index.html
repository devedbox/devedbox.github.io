#小工蜂业主端开发规范

版本纪录

版本 修改人员 日期
1.0.0 艾星 2016年3月19日

说明

此文档用于指导服务器拟定API交付文档以及客户端开发规范，在实际开发中，客户端可根据实际情况做响应的调整，调整的内容需要在此文档做出相应的更改和增删。服务器在拟定开发文档若有本文未涉及到的字段命名或定义，需及时提出增加或修改意见，一切字段的命名或者修改需要做好相应的纪录，避免后期开发调试过程中出现不该出现的因为字段命名而产生的错误！

命名空间

GFPropertyOwner缩写：GFPO.

通用模型

网络请求通用描述语：

  • request- 请求完成一个操作，涉及到服务器有逻辑处理的成分在里边.
  • get- 请求数据信息，比如请求维修统计数据、请求验证码.
  • update- 更新信息，涉及到服务器更新数据库操作.

此处的描述语更多的是在语意上通俗的表述一个操作，比如请求下单的时候，涉及到服务器的逻辑、数据库操作比较多，但是更新信息同样也涉及到了同样的操作，为什么更新信息用update而不用request呢？其实都可以，但是对比一下requestOrder:和updateOrder:在语意上request更接近我们所要表述的意思.

字段描述

Name

字段定义.

Defination

字段中文描述.

Type

字段对应的数据类型，大致分为两种：

number：表示使用int、long、double，具体类型在服务器给定的API文档中指定，在客户端研发的过程中根据各自平台选择相应的适合的数据类型.

string：表述使用字符串类型.

object：指代对象类型，在相应的地方会进行相应的指定.

Value

字段定义值，针对enum类型.

进行中的订单:订单 GFPOProcessingOrderObject

进行中的订单继承于普通订单，在普通订单的基础上，还拥有以下属性：

Name Defination Type
updationTime 订单状态更改时间 number
state 订单状态 number
substate 订单子状态 number
masterName 师傅名字 string
masterPhone 师傅电话 string

其中，updationTime是每次订单状态更改都会更新的时间属性，在业主端客户端用来进行排序.

历史订单：进行中的订单 GFPOHistoryOrderObject

历史订单同样继承于进行中的订单，在进行中的订单的基础上，多了以下属性：

Name Defination Type
isPayed 订单是否支付 boolean
isCommented 订单是否评价 boolean
comment 评价内容 GFPOCommentObject
cost 支付价格 number
